## 🔑 First Time Setup

### Step 1: Initialize Git Repository

If you haven't already initialized git:

```bash
cd election-assistant

# Initialize git (one-time)
git init

# Add remote repository
git remote add origin https://github.com/YOUR_USERNAME/election-assistant.git

# Verify remote
git remote -v
```

### Step 2: Create .gitignore

The `.gitignore` file tells Git which files to ignore. Create it:

```bash
cat > .gitignore << 'EOF'
# Dependencies
node_modules/
npm-debug.log
package-lock.json

# Environment variables (NEVER commit secrets!)
.env
.env.local
.env.*.local
.env.example (after you've copied it)

# IDE
.vscode/
.idea/
*.swp
*.swo
*~
.DS_Store

# Build outputs
dist/
build/
out/

# Logs
logs/
*.log
npm-debug.log*

# OS
.DS_Store
Thumbs.db

# Production
deploy/
EOF
```

### Step 3: Initial Commit

```bash
# Add all files
git add .

# Commit
git commit -m "Initial commit: AI Election Assistant v3.0.0 - Full-Stack"

# Push to GitHub
git branch -M main
git push -u origin main
```

---

## 📝 .gitignore Explanation

### Never Commit
```
.env                    ← Contains database password & secrets
node_modules/           ← Generated by npm install
*.log                   ← Log files
.vscode/                ← Personal IDE settings
```

### Safe to Commit
```
package.json            ← Dependency list (version locked)
package-lock.json      ← NO! This is environment-specific
backend/models/         ← Database schemas
backend/routes/         ← API routes
```

## 🚀 Deployment via GitHub

### For Vercel (Frontend)

1. **Connect to Vercel**
   ```
   1. Go to vercel.com
   2. Click "New Project"
   3. Import from GitHub
   4. Select your repository
   5. Select "Other" as framework

   ```

   ```
   2. Click "New" → "Project from GitHub"

   4. Select "backend" folder

   ```

2. **Auto-Deploy on Push**
   ```
   Any push to 'main' branch automatically deploys
   ```

## 🔒 Protecting Secrets

### What to NEVER Commit

```
✗ Database passwords
✗ JWT secret keys
✗ API keys
✗ Email credentials
✗ OAuth tokens
✗ Any sensitive data
```

### How to Protect

1. **Use .gitignore**
   ```
   # Don't commit environment files
   .env
   .env.*.local
   ```

2. **Use .env.example**
   ```
   # This shows what variables are needed
   # But without actual values
   MONGODB_URI=
   JWT_SECRET=
   ```

3. **Set Secrets on Deployment Platform**
   - Vercel: Settings → Environment Variables
   - Railway: Variables tab
   - GitHub: Settings → Secrets

## 📊 Repository Best Practices

### 1. Meaningful Commit Messages
```bash
# ✓ Good
git commit -m "Add rate limiting middleware"
git commit -m "Fix: admin dashboard loading issue"
git commit -m "Refactor: improve error handling"

# ✗ Bad
git commit -m "updates"
git commit -m "fix stuff"
git commit -m "wip"
```

### 2. Atomic Commits
```bash
# ✓ Good - One logical change
git commit -m "Add vote counting API endpoint"

# ✗ Bad - Multiple unrelated changes
git commit -m "Add voting, fix styling, update docs"
```

### 3. Branch Naming
```bash
feature/admin-dashboard      # New feature
bugfix/vote-counting         # Bug fix
docs/deployment-guide        # Documentation
refactor/auth-middleware     # Code improvement
```

### 4. Pull Requests
```
1. Push feature to new branch
2. Go to GitHub
3. Click "Compare & pull request"
4. Add description of changes
5. Ask for review
6. Merge when approved
```

## 🔄 Continuous Integration (CI)

### Add GitHub Actions (Optional)

Create `.github/workflows/test.yml`:

```yaml
name: Tests

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - uses: actions/setup-node@v2
      with:
        node-version: '16'
    
    - name: Install dependencies
      run: cd backend && npm install
    
    - name: Check for errors
      run: cd backend && npm test
```

## 🆘 Common Issues

### Issue: "Permission denied (publickey)"
```bash
# You need to set up SSH key
# Follow: https://docs.github.com/en/authentication/connecting-to-github-with-ssh

# Or use HTTPS instead:
git remote set-url origin https://github.com/YOUR_USERNAME/repo.git
```

### Issue: "Everything up-to-date" but changes not pushed
```bash
# Verify you committed changes
git status                      # See uncommitted files
git add .
git commit -m "Your message"
git push
```

### Issue: "Accidentally committed .env file"
```bash
# Remove from git history (careful!)
git rm --cached .env
echo ".env" >> .gitignore
git commit -m "Remove .env from tracking"
git push

# IMPORTANT: Regenerate any secrets exposed!
```
